# API & OpenAPI Patterns

## Items

<!-- @item source:user status:active enforcement:strict -->
Use Pydantic models for all API request/response schemas. This ensures automatic OpenAPI documentation and runtime validation.

<!-- @item source:user status:active enforcement:locked -->
Never use `dict` as an API response type. Always define a Pydantic model, even for simple responses.

<!-- @item source:user status:active enforcement:strict -->
After adding or modifying API endpoints, run `make sync_openapi` to regenerate frontend types.

<!-- @item source:user status:active enforcement:locked -->
Files in `frontend/src/openapi/` are auto-generated by Orval. Do not edit manually.

<!-- @item source:user status:active enforcement:strict -->
Register new route modules in `app/api/router.py`.

<!-- @item source:user status:active enforcement:strict -->
**Async route handlers**: Always use `async def` for FastAPI route handlers. This ensures consistent behavior and prepares the codebase for async database operations. Example: `async def get_items(user: UserDependency):`
